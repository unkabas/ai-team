# ai-team — файловая шина для координации ИИ-агентов

Шаблон-репозиторий (не пакет и не лаунчер): **форкаешь → клонируешь → работаешь**.
Несколько ИИ-агентов в разных терминалах ведут одну кодовую базу **без перезаписи
файлов друг друга и без дублирования логики**. Координация — обычная директория
`.agent_workspace/` плюс CLI `./bus` (bash + jq, zero-install).

## Почему это работает

- **Атомарные локи.** Любое изменение `state.json` сериализуется mkdir-мьютексом
  (атомарный syscall) + запись через temp-файл и `mv`. 20 агентов параллельно —
  ноль порчи состояния, ноль двойных захватов (проверено).
- **Lease-локи.** Задача авто-освобождается, если захвативший агент умер
  (`./bus doctor --fix`). Не нужна дисциплина — за целостностью следит CLI.
- **Правило «критикуешь — предлагай»** зашито в синтаксис: `./bus cr` отклоняет
  критику без альтернативы. Команда спорит по технике и находит лучший вариант.
- **Экономия токенов.** Машинный слой пишется компактным `buslang`
  (см. `.agent_workspace/buslang.md`); человеку — отдельный читаемый слой
  `ideas.md` / `decisions.md`.
- **Без лимита агентов.** Сколько терминалов запустишь — столько и работает.

## Требования

- `bash` (совместимо с 3.2, штатный для macOS) и `jq`.
  macOS: `brew install jq`. На macOS `flock` не нужен — лок через `mkdir`.

## Быстрый старт (склонировал → написал цель → дальше сами)

```bash
git clone <твой-форк> && cd ai-team     # или скопируй bus, lib/, .agent_workspace/ в свой проект
./bus init                              # workspace + автодетект стека + Next steps
./bus goal "сделать REST API для заметок"   # просто опиши, ЧТО хочешь
./bus agent alice                       # запусти в N терминалах (см. ниже)
```

После `goal` ручное наполнение задач не нужно: первый запущенный агент видит,
что backlog пуст, а цель есть — **атомарно** захватывает планирование
(`plan-claim`, выигрывает ровно один), разбивает цель на 3-7 непересекающихся
задач и заводит их. Остальные агенты разбирают backlog, делают, ревьюят и спорят
по правилу «критикуешь — предлагай» — без твоего участия.

`./bus init` сам сканирует проект (package.json, go.mod, Cargo.toml,
requirements.txt, pom.xml, Gemfile, *.csproj ...), определяет стек и команду
тестов и вписывает их в `context.md` (между маркерами `BUS:AUTODETECT`, ручной
текст не трогается).

## Запуск команды агентов

Открой N терминалов. В каждом:

```bash
./bus agent alice    # печатает готовый init-промпт с подставленным именем
```

Скопируй вывод в сессию агента — дальше он сам: при старте делает `plan-claim`
(вдруг он первый — тогда планирует), затем берёт задачи через `./bus`. Имена
уникальные (alice, bob, ...), число агентов не ограничено.

> Сменил цель на ходу? `./bus goal "новая цель"` сбрасывает флаг — следующий
> свободный агент перепланирует. `./bus goal` без аргумента покажет текущую цель.

## Файлы workspace

| Файл | Роль | Слой |
|------|------|------|
| `context.md` | архитектура, стек, стандарты, Source-of-Truth index | человек |
| `state.json` | менеджер задач + lease-локи (машинный, через `./bus`) | машина |
| `event_log.md` | append-журнал действий (buslang) | машина |
| `communication_bus.md` | вопросы/ответы/критика между агентами (buslang) | машина |
| `error_dump.md` | трейсы падений (чтобы не зацикливаться на баге) | машина |
| `buslang.md` | спецификация сжатого языка | справка |
| `agent_prompt.md` | init-промпт агента | справка |
| `ideas.md` | открытые идеи команды | человек |
| `decisions.md` | журнал принятых решений | человек |

## Команды `./bus`

```
init                                  развернуть workspace + автодетект стека + Next steps
goal "<что хочу>"                     задать цель (агенты сами разложат на задачи)
agent <NAME>                          готовый init-промпт для запуска агента (копипаст)
status [--json]                       задачи по статусам + кто держит лок
add-task <title> [desc]               добавить задачу в backlog
claim <agent> [task_id]               захватить задачу (первую свободную если без id)
move <agent> <task_id> <status>       перевести свою задачу (in_progress|review|done|backlog)
release <agent> <task_id>             снять свой лок (задача -> backlog)
log <agent> <file> <result...>        запись в event_log.md
error <agent> <message...>            запись в error_dump.md
ask <agent> <target> <q...>           вопрос в communication_bus.md
answer <agent> <ref> <text...>        ответ
cr <agent> <target> <task> --crit "..." --prop "..."
                                      критика С предложением (иначе отказ)
doctor [--fix]                        проверка целостности; --fix реклеймит протухшие
```

## Дорожная карта (итерация 2)

`./bus next` (выдача задачи + нужных файлов одной командой), ownership-map по
файлам в `claim`, lifecycle статусы сообщений + `./bus inbox`, авто-handoff при
`release`.
